# Configuration

Auth2Demo uses ASP.NET Core configuration. Values may come from JSON files, user secrets, environment variables, or a production secret provider.

## Connection string

Required key:

```text
ConnectionStrings:DefaultConnection
```

Environment variable form:

```text
ConnectionStrings__DefaultConnection
```

## Local user secrets

From the Web project directory:

```powershell
dotnet user-secrets init
dotnet user-secrets set "ConnectionStrings:DefaultConnection" "..."
```

Do not use user secrets as a production secret store.

## External identity providers

Provider records are stored in the database and scoped to a company. Typical configuration includes:

- provider type;
- display name;
- authority or tenant identifier;
- client ID;
- protected client secret;
- enabled state;
- optional claim mapping and visual metadata.

The callback URLs registered at Google or Microsoft must exactly match the URLs generated by the deployed Auth2Demo instance.

## Provider secret protection

`IIdentityProviderSecretProtector` protects provider credentials before storage. Production deployments should protect the underlying key material with a platform secret manager or certificate and must share that key material between application instances.

## Cookies

Current application-cookie configuration:

```text
Name: Auth2Demo.Identity
HttpOnly: true
SecurePolicy: Always
SameSite: Lax
SlidingExpiration: true
```

Review cookie domain, SameSite mode, lifetime, and consent behavior for the final topology.

Culture selection order is documented in `localization.md`.

## Forwarded headers

`X-Forwarded-For` and `X-Forwarded-Proto` are enabled. The current code clears known proxy/network restrictions, which is convenient for development but should be tightened in production to trusted reverse proxies and networks.

## Logging

Use structured logging and avoid logging:

- passwords;
- client secrets;
- provider secrets;
- authorization codes;
- refresh tokens;
- access tokens or ID tokens;
- full security answers or recovery codes;
- sensitive claims not needed for diagnostics.

## Environment-specific configuration

Recommended files and sources:

```text
appsettings.json                 non-secret defaults
appsettings.Development.json     local development defaults
User Secrets                     local credentials
Environment variables            container and CI/CD values
Key Vault / secret manager       production credentials and keys
```
